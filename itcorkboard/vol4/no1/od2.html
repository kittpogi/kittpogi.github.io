Office of the Director : Updates

On-line Payment
by: Samuel P. Mallare

We have a three-fold electronic payment (e-payment) initiative. One is on-line (meaning, web-based) payment via credit card. We are coordinating with Equitable Card Network. Another is on-line payment via ATM card. We are coordinating with Yehey! We coordinated with a number of providers but there were concerns (e.g., high processing fee and poor feedback) that resulted to loss of interest in pursuing the initiative with them. The last is the pre-paid or debit system using the employee/student ID card. Employees and students can pay, for example, P500 at the Accounting Office. This can be used for small payments for parking, library surcharges, and printing. These should be enough to automate payment transactions, shorten or eliminate queues, improve processes, and ensure payments are in proper order.

One option that we are not pursuing is the recording of data, like updated balance (or "credit"), on the smart card. This is what some vendors offer. They call this e-purse or e-wallet. With this, the cardholder can transact payments with establishments other than DLSU since the "credit" is recorded on the card. Other information, like medical details, can be stored on the smart card as well. The ID can even be combined with the ATM card of a bank. However, this would require more coordination on the information systems crossing organizational boundaries. One implication is that the payments for DLSU will be transferred to DLSU bank accounts after one or a few banking days.

For e-payments, we need to educate ourselves and our customers on the security of e-payment. This is to increase our confidence level on using such technologies. We have coordinated with MCO, Alumni Association, and Student Publication Office to help us disseminate articles about e-payment security.

Automated Software Distribution
by: Samuel P. Mallare

This will enable CFO to install software packages in a pull and/or push mode. Right now, CFO is able to do this only with the virus signature. The anti-virus program sends out updates on virus signatures as soon as they are available from the vendor. In our case, the vendor is Data Fellows (Europe).

We have evaluated a number of products based on information from the web and product brochures. We will be doing a proof of concept (i.e., demo, evaluation, and testing) with the Unicenter Software Deploy of Computer Associates. We hope to get the approval of the administration on this project.

The DLS System campuses are also interested to implement these projects. The IT Directors are coordinating among themselves on the enterprise portal and automated software deployment.

Right now, only PSI, Manila, and Dasmariñas have their portals. Dasmariñas has already incorporated a web-based discussion board in their portal. Everyone is excited to implement a portal package considering the benefits the entire System will get. The portal would provide a good software infrastructure for a virtual university. This is one of the long-term projects that we can work on. If DLSU plans to offer full on-line degree programs, we should be ready with the hardware and software, infrastructure, support services, and appropriate policies and procedures.

All campuses have their way of installing software packages. All agree that this is one of the most tedious tasks in IT operations. Automating this process can reduce, if not eliminate, the threat of security and virus attacks. This could also eliminate our weakness of not being able to immediately service urgent needs on software installation and re-installation.

There is still a lot we can achieve. The primary framework for our software development initiatives is the application service provider (ASP) model where web-based information systems are developed with object-oriented technologies. This will enable us to service many, if not all, members of the DLS System.

The initial challenge we face is the variety of data - e.g., ID number, course code, and information (or report) requirements. We will be able to transform these challenges into achievements by first accepting that our role in the System has evolved and we can contribute much more to the achievement of the DLS System mission-vision.
