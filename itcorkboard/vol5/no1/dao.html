Database Objects Naming Convention
by: Florinda D. Tamonan

Any development effort requires consistency for success. With this in mind, the Data Administration Office created a database object and application naming convention for SQL and Oracle Database. Our goal is to make a set of guidelines to achieve uniformity throughout all systems being developed and to make DBA, developers’ tasks easier and faster. These “standards” also aim to limit the ambiguity and uncertainty surrounding an object and its purpose.

The guideline includes the general set of rules for each object in the database. The documentation also includes helpful Appendices. Appendices I to II contain the existing field name and field abbreviation. Appendices III to IV speak of standards for tables and views while Appendices V to VI include the developers’ coding formats.

Traditionally, developers and administrator uses different approaches and methodologies making it harder to manage and maintain program codes and data but we hope to alleviate the problem through this standards we can say “the best practices”.

Integrated Department File
by: Julio Ceres R. Ibay

The Data Administration Office (DAO), together with the Systems Management Office (SMO) is currently in the process of implementing the use of the newly integrated department file. DAO completed the mapping of files to be used by SMO in replacing the department files in the different application systems that they are maintaining, with the objective of achieving consistency in data and a more comprehensive department file. SMO is scheduled to complete the transition process within the first term of the current school year.

School Masterfile Update
by: Julio Ceres R. Ibay

The School File Update request will be available on-line. Requests for additional schools as well as changes on existing schools in the SRM database can be entered directly in the School Masterfile Maintenance. The user need not email the School Update Request form but will still have to send a SRF The request will be validated before the corresponding codes will be assigned or other changes will be implemented. The Data Administration Office is close to completing the said program and is projecting its implemenation before the end of 1st Term, School Year 2003-2004. This will facilitate faster and more efficient processing of school update requests.
