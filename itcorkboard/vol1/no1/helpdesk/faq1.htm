   Frequently Asked Question

 

 Do you ever have to spend some grueling hours over some irregularities in the use of your Internet Account? Has it also ever crossed your mind that there must be a way to mitigate this unforeseen situation such that it gives you direction and save your time. If so, you are on the right track. Some commonly encountered problems are compiled here to provide the users with insights on the nature of these problems and right approach in handling these problems. So the next time you encounter any of the errors listed below, you would not feel empty handed, instead you can take immediate steps in getting your Internet Account working for you. Keep yourself posted for any update.

  Problem Set 1   LOGIN ERRORS


1.1  Access denied or cannot login to your account

Solution: Here is a checklist of things to ask yourself

    1. Are you the rightful owner of the user account you are trying to logon to?
    2. Is your computer connected to the network?
    3. Are you using the correct username? password? login procedure?

If your answer is yes to all these questions and you still cannot gain access to your account, contact the ISTC HelpDesk at local 316 or direct line number 526-4242.

1.2 An error message – "LOGIN-4.13.830. You are trying to login to too many stations simultaneously. The supervisor has limited the number of user connections you may have" – appears on screen.

Cause: The user has failed to log out from his account and tries to login to his account using another terminal.

Solution: Call ISTC and request your last login connection to be terminated.

1.3 An error message – "Your current context is istc.dlsu-manila. The user specified does not exist in this context. Login will try to find the user in the server context. dlsu-manila. The context istc.dlsu-manila is invalid. LOGIN-4.13-100: Access has been denied." – appears on screen.

Solution:
Option 1

  • When you login through MS-DOS prompt, type "cx/r" in order to go to the root directory. Press Enter. Type the complete login syntax (refer to the Login Procedure under HelpDesk Corner in Tech Nook).

Option 2

  • When you login through the Novell login box (Novell Client for Windows 95), click on Advanced. Fill out the fields in NDS. For Netware6 users, enter "dlsu-manila" under Tree. In Context, enter "[group name].dlsu-manila",e.g., istc.dlsu-manila. Other group names are adm,sps,gsb,fsc,cos,cla,ced,coe,office,org, reg, etc. For Mail2 and Mail3 users, the Tree and Context fields are left blank. Choose the right server (e.g. Netware6, mail2, mail3).

1.4 An error message – "LOGIN-4.12 (950418) PTF-100: Access has been denied." – appears on screen.

Cause: The Personal Computer is connected by default to a server in which you are not listed as an authorized user.

Solution:
Option 1

  • When you login through MS-DOS prompt, type the complete the login syntax including your required server connection. Also, refer to the Login Procedure.

E.g., F:\LOGIN> login mail3\09876543.ed98.mail3

Option 2

  • When you login through Novell login box, check your server type in NDS and make sure that you are connected to the right server.

  Problem Set 2   PEGASUS MAIL PROGRAM ERRORS


2.1 Using Pegasus Mail, there are garbled messages on your screen, or a recurring end-of-line messages on screen, or if you are unable to delete / read e-mail.

Solution: In the DOS prompt, make sure that you are in your home directory. i.e., Netware6 F:>\user\istc\cscldm> [type]repair. This will  run the "repair" program.

2.2 Encounter a server queue error when sending an e-mail.

Cause: The system interprets that there is no more free disk space in your account so it can no longer delete or send e-mail. In order to verify if there is no more disk space available, go to home directory and type "dir". It should indicate zero free byte.

Solution: Try to delete unnecessary files and trim down your folders. In your home directory, type "purge/all". This command will permanently remove all deleted files from the storage bin. Logout and turn off the computer. Wait for two minutes before logging in to your account and run "repair". Check if you can already send an e-mail. If not, call ISTC.

2.3 How to recover deleted file/s in your mailbox.

Solution:

  1. In your home directory, run the filer program by simply typing "filer".
  2. In Available Options, highlight salvage deleted files and press enter.
  3. The salvage menu box will be shown. Choose view / recover deleted files.
  4. In the erased file name pattern to match, specify the filename (only if you can remember). Otherwise, leave the asterisk by default and press enter.
  5. Go to PMAIL <DIR>, press F5 to mark the file/s that you want to be retrieved.
  6. Under recover this file, answer yes. Finally, check the recovered file/s in your mailbox.

2.4 Encounter an intruder lockout detection error message on screen.

Cause: Attempt to login to one's account unsuccessfully for more than three times.

Solution: Wait for three minutes before trying to login to your account again. If access is denied, call ISTC to reset your password and to check your intruder lockout status.
