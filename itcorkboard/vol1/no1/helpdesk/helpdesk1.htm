   Help Desk Corner


Has anybody ever sent you an e-mail with file attachment? If you are a regular e-mail user, you would probably not have difficulty in opening and reading the attached file. You would know right away that the first step is to extract the file by pressing key x. You may specify the path (either the disk or hard disk drive, eg., C:\WINDOWS\[filename]) or, store the data to the network drive, by default, if you fail to indicate a path.

You are now ready to open the extracted file using any browser tool such as MSWord, MSExcel, MSPowerPoint, Netscape Communicator, etc. The choice of what browser tool to use would depend on the file type. This is usually evident in the extension file name (eg., The filename can be Voltes.doc for MSWord, or Voltes.txt for text file, or Voltes.ppt for MSPowerPoint file).

PKZIP / PKUNZIP or ARJ

Supposing you have done this process a couple of times but suddenly, it does not work as expected. What can possibly be wrong? If you are not aware of PKZIP / PKUNZIP or ARJ, you would likely feel helpless by yourself. Agonize no more. This article presents an overview of what it is all about and how it works.

PKZIP / PKUNZIP and ARJ are file compression utilities. This way the file size can be temporarily reduced, thereby, allowing large volume data to fit in one diskette, or in multiple diskettes. Aside from the space saved in a diskette or hard disk, it also results to faster mail delivery in view of a lighter load. An ARJ or ZIP file can easily be recognized through its extension file name, e.g., Voltes.arj or Voltes.zip.

These compression tools are usually installed in the personal computer’s drive C under the DOS directory. However, even if these are not found in that directory, you would still be able to run these programs in any drive provided that you are logged in to the network.

PKZIP and ARJ actually create a file where several files or one whole directory can be condensed and archived into. Just type ARJ or PKZIP in MS-DOS prompt and its usage will be displayed on screen as shown below:

PKZIP [option] [zipfile] [files you wish to compress]

Example: PKZIP -exrp leo.zip a:/windows/*.*

  • ex is to set the compression speed mode to extra.
  • r is to automatically include subdirectory files in the zipfile.
  • p is to store pathname.

PKUNZIP [option] [zipfile] [files you wish to decompress]

Example: PKUNZIP –esd leo.zip a:/windows/*.*

  • es is to sort the extracted files by size.
  • d is to restore directory contained in zipfile.

ARJ [option] [archive name] [file_names]

Example: ARJ -a leo.arj a:/windows/*.* -va –jm1

  • a is to archive files.
  • va allows the use of multiple 1.44 MB diskettes.
  • jm1 is for maximum compression at the fastest speed.

Other options for ARJ are as follows:

  • e is to extract file/s.
  • ve allows the use of multiple 1.2 MB diskettes.
  • je is to create an executable compressed file.

Note: Those in * are some useful options. For other options, press 2 using PKZIP / PKUNZIP and ARJ /? using ARJ.

In summary, when you receive a ZIP or ARJ file as an e-mail attachment, you have to eXtract the file and expand it in order to read the file using the appropriate browser tool.

UUDECODE / UUENCODE

Aside from a ZIP or ARJ file attachment, you must also have encountered another file attachment which is not in a regular format, and has garbled messages on it. An onscreen message explicitly tells the recipient that the mail has been UUENCODED. At the same time, the "begin" header appears at the start of the text body and, the "end" is marked as well.

You may convert this message to a readable file using the UUDECODE program. The steps are outlined below:

1) Extract the message and save it as text file.

2) Using MSDOS Editor, open the extracted text file by simply typing "edit [file name]".

E.g., C:\WINDOWS> edit Voltes.txt

Delete part of the text that lies outside the "begin" and "end" note. Only those characters that are delimited by the "begin" and "end" are convertible to a readable text.

3) Save the edited file. Again, using MSDOS Editor, press Alt-F and choose Save.

4) UUDECODE and UUENCODE programs are installed in the network drive under U:\apps\freeware. To covert the file from binary to original format, type

UUDECODE [UUENCODED file_name] [output file_name]

E.g., F:\user\csc\cscldm> U:\apps\freeware\uudecode Voltes.txt Leo.txt

5) Finally, read the message using by using the appropriate browser program.

On the point of view of the sender, use UUENCODE to convert the file to a binary text in transmitting the message. So there you go folks! File attachments can be in a regular format, or ZIP or ARJ file, or UUENCODED file. Whatever the case may be, you should know better what to do with each one after reading this.
