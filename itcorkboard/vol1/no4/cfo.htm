DLSU Installs more Network Access Ports

To increase the access of the academic community to the Internet, network access ports (NAPs) have been installed around the campus.  Students or faculty members can bring their notebook computers with network interface cards (NIC) and patch cords, and connect to the Internet through the access ports.  The procedure on how to use the NAPs is available in the Help Desk web site at http://mail.dlsu.edu.ph/~helpdesk

 To date, there are 243 NAPs installed.  The locations of these are as follows:

  Location of Units   Number
  Medrano Study Hall   50
  L218 Case Room   43
  L219 Case Room    50
  Gokongwei Classrooms (2nd floor)   12
  Gokongwei Classrooms (4th floor)   6
  CCS Project Rooms (G409, G410, G411)   8 each
  Gokongwei Lobby   16
  STRC412 Seminar Room   42

Dial-up Lines to be Increased

ISTC has requested new telephone lines to increase dial-up services to DLSU.  The increased number of lines will facilitate the remote access to the network by students and faculty members.  So far, fifty (50) lines were approved by the DLSU Administration.  The Physical Facilities Office will coordinate the installation of the requested telephone lines with Eastern Telecoms.  

ISTC to Acquire a Remote Access Server

ISTC plans to acquire a Remote Access Server (RAS) to mount several modems on a single chassis and manage these from a single console.  This will greatly facilitate connection, thus minimizing time online and maximizing the number of transactions completed.  ISTC is currently testing the demo unit provided.

DLSU’s Preparation for the Coming of the Bug

As early as 1997, preparation for the Y2K bug has started at DLSU.  All equipment in campus, laboratory, audio-visual, communication, computer and software system have been checked for compliance.  ISTC installed patches for Windows 95/NT and Netware.  Suppliers were requested to submit certification of their products for Y2K compliance.  For existing computers which were not Y2K-compliant, such as 486s, users were taught how to change the date in the BIOS.  For the higher models of computers, such as Pentiums, these were found to be Y2K-compliant.

With respect to the application systems, ISTC-Systems Management Office conducted an inventory of all programs to check compliance.  Several systems that would not run by year 2000 were identified.  Some Faculty Information System reports affected were revised.  The Student Records Management running on VAX-RDB was migrated to the Oracle system.  Faculty Attendance Monitoring in Clipper was revised using Visual FoxPro.  The De La Salle-Funds Accounting System was replaced by the Oracle Financials-GL/AP/AR.

Celine V Rivera, 02/11/2000
